# Global Pay - Contributing Guidelines

## How to Contribute

We welcome contributions from the community! Here's how you can help:

### Reporting Bugs
1. Check if the bug has already been reported
2. Create an issue with a clear title and description
3. Include steps to reproduce and expected behavior
4. Add relevant screenshots or logs

### Suggesting Enhancements
1. Use GitHub Issues with a descriptive title
2. Provide use cases and expected behavior
3. Explain why this enhancement would be useful

### Pull Request Process

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes following the code style
4. Write or update tests
5. Commit with clear messages: `git commit -m 'Add feature: description'`
6. Push to your fork: `git push origin feature/your-feature`
7. Create a Pull Request with a descriptive title and body

### Code Style

- **Java**: Follow Google Java Style Guide
- **TypeScript/React**: Use ESLint and Prettier configurations
- **SQL**: Use UPPER CASE for keywords, snake_case for identifiers

### Testing

- Write unit tests for new features
- Ensure all tests pass: `mvn test` (backend), `npm test` (frontend)
- Maintain code coverage above 80%

### Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new feature description`
- `fix: Fix bug description`
- `docs: Update documentation`
- `style: Format code`
- `refactor: Refactor code`
- `test: Add tests`

### Branch Naming

- Feature: `feature/short-description`
- Bugfix: `bugfix/short-description`
- Documentation: `docs/short-description`

## Code of Conduct

- Be respectful and inclusive
- No harassment or discrimination
- Report issues to the maintainers

## Questions?

Feel free to open an issue or contact the maintainers.
